Module 0 - Introduction to Calcium imaging

Calcium imaging is used in neuroscience as a proxy for neural activity, because calcium acts as a secondary signaling molecule for neural function.

There is significant flexibility with a growing list of methods across the chemical space of calcium binding, optical space of calcium imaging, and software space of signal processing. While this flexibility is helpful for advanced users, it can create a steep learning curve for new users. This tutorial is designed to help smooth that curve with practical, modular introductions.

0.1 How does calcium imaging work?

Grienberger and Konnerth provide a strong review of biological calcium signaling and practical imaging approaches. It is a good starting point for calcium indicators (chemical and genetically encoded), dye-loading techniques, and imaging modalities.

Calcium imaging requires:

  1. Host tissue, typically neurons in live animals or cultured cells.
  2. Calcium-sensitive sensors. Genetically encoded indicators can be delivered with non-replicating viral vectors or transgenic lines. Chemical sensors (for example Fluo, Oregon Green, and Calbryte) are also used, but are usually limited to short-duration experiments (< 6 hours) and are less common for in vivo long-term work.
  3. Optical systems that excite and detect fluorescence, including one-photon or two-photon imaging.
  4. Signal processing pipelines to extract and interpret fluorescence changes from imaging videos.

0.2 Understanding calcium signals in neurons

Ali and Kwan explain differences in in vivo calcium signals from soma, axons, and dendrites, and discuss how to interpret these signals relative to behavior.

This module primarily focuses on somatic calcium imaging:

Somatic calcium imaging can provide insights into neuronal activity because voltage-gated calcium channels permit rapid influx into the soma during action potentials.

Dendritic calcium imaging is also important because local voltage changes and synaptic input integration create local calcium signals in dendritic compartments.

Side note: this tutorial covers only a subset of what calcium imaging can do. Calcium readouts are also used to track changes in neuronal function under mechanical forces, astrocytes, osteocytes, and plants.

0.3 Why do we care about calcium and neural dynamics?

The cerebral cortex is essential for behavior, including sensory processing, voluntary movement, and higher-order cognition. These functions emerge from local and global circuit dynamics.

In systems neuroscience, one major goal is to identify the circuits that establish behavior. This requires tracking activity relative to behavior, and then probing or modulating components of the circuit. Recording alone usually gives limited causal insight.

There is a growing toolbox across circuit features and recording modalities. Electrode approaches (EEG, ECoG, and shank recordings) can be powerful, but are often limited in single-cell spatial resolution. Optical methods such as calcium imaging and voltage imaging provide cell-resolved access to large populations, which makes them especially useful for studying circuit-level organization.

In cortex, coordinated firing groups are often described as ensembles. Calcium imaging allows direct visualization of these population motifs and their relationship to behavior. For broader context on why systems neuroscience benefits from moving from single-neuron analysis to network analysis, see Yuste’s review.

Module 1 - Exploring somatic calcium dynamics

Learning the full calcium processing pipeline can be overwhelming at first. In this module, you will start with extracted calcium traces and build your own processing scripts.

Calcium signal processing is commonly done in MATLAB or Python. The field is increasingly using Python, but MATLAB is still widely used in many labs.

If you want a broader methods review for neural coding from calcium imaging: Saito et al. is a strong reference.

Task 1.0 - Importing data

For this module, traces are already extracted from calcium video data. In later modules we will cover extraction pipelines directly (for example, Suite2P).

  • Dataset shape: 324 neurons x 15000 frames.
  • Values: raw mean fluorescence intensity from GCaMP6s-expressing neurons.
  • Sampling rate: 4.8 frames per second.

Task 1.1 - Signal cleaning

Goal: clean raw signals so calcium transients can be detected robustly.

Step 1.1.0 - Normalize the signal

Compute ΔF/F as percentage change from baseline: ΔF[t]/F = (F[t] - F0) / F0

where F0 is resting calcium intensity when no firing occurs. Hint: when event times are unknown, a lower quantile estimate can be useful for F0.

Step 1.1.1 - Subtract background noise

Remove slow (> 20 second) intensity fluctuations caused by optics or baseline drift while preserving short-timescale transient events.

Step 1.1.2 - Denoise signal

Reduce high-frequency noise variance without distorting event shape. Wavelet denoising is one common approach.

Deliverable: plot representative traces before and after each step (raw, normalized, background-subtracted, denoised).

Task 1.2 - Detect transient calcium events

Goal: identify time points with rapid calcium influx.

Important distinction: calcium event is not strictly equal to a neuron spike. Events are strongly related to spiking, but not one-to-one.

For spike deconvolution context, see: algorithms for inferring spikes from calcium traces.

For this module, prioritize calcium event detection. Typical single GCaMP6s events often have FWHM < 5 seconds, though repeated spikes can stack and broaden transients.

  • Start with neuron-specific thresholding (for example, signal > mean + 5 SD).
  • Add a second discrete threshold if low-activity neurons create false positives.
  • Optional challenge methods: template matching or wavelet ridgewalking.

Deliverables:

  1. Plot example calcium traces with detected event locations highlighted.
  2. Create a scatter/raster plot (x: time, y: neuron ID, points: detected events).
  3. Extract transient metrics (for example spike rate summaries and custom metrics).

Task 1.3 - Extracting network features

Goal: detect neuronal ensembles, meaning groups of neurons with coordinated activity in short windows. The specific time window where a candidate group fires is an ensemble event.

Since ensemble identities are not known beforehand, start by detecting significant ensemble events, then group similar events into recurring ensembles. For conceptual and analytical background, see Wenzel and Hamm.

Step 1.3.0 - Compute neuronal coactivity

Neuronal events are not perfectly synchronous, so bin events in short windows (for example 500 ms), then count active neurons per bin as a coactivity signal.

Deliverable: coactivity vs. time plot showing high-activity periods.

Step 1.3.1 - Identify ensemble events

Build a null distribution by circularly shifting each neuron’s event train (random cut point) so autocorrelation structure is preserved while physiological coactivity is disrupted.

Repeat shuffling (for example 1000 shuffles), recompute coactivity each time, and compare real coactivity to the null distribution. Mark events above the 99th percentile as significant ensemble events.

Step 1.3.2 - Identify ensembles

Not all ensemble events represent distinct ensembles, because biological variability can shift participation. Compute similarity between contributor sets for each ensemble event, then cluster similar events.

Use pairwise Jaccard similarity between binary contributor vectors: S[x,y] = jaccard_similarity(ensemble_event_x, ensemble_event_y)

Visualize the similarity matrix as a heatmap, then group events with clustering (for example hierarchical clustering). PCA-informed cutoffs can help select cluster depth.

Deliverables:

  1. Present identified ensembles and associated ensemble events.
  2. List which neurons contribute to each ensemble.
  3. Describe when each ensemble is active in the recording window.
