Why this guide starts with populations

Single neurons rarely tell the whole dynamical story

Single neurons can be highly informative, but they are often noisy, mixed, and context dependent. In the classic work of Shadlen and Newsome, trial-to-trial variability helped make clear that the mechanism behind a computation is often easier to see at the population level than in one spike train at a time.

A dynamical-systems view shifts the question from "what does this one neuron encode?" to "how does the full neural state evolve?" That perspective aligns with later population-level work showing that decision and motor computations often appear as structured trajectories through a lower-dimensional state space rather than as static tuning in isolated cells, as emphasized by Churchland et al..

The pendulum below is a clean toy system for this shift in perspective. On the left you see motion in physical space. On the right you see the same process rewritten as a state evolving through a latent space. That left-right translation is the same conceptual move used in neural data when we infer dynamics from population trajectories rather than from one neuron alone.

Module 0 - What is a dynamical system?

A dynamical system is a rule for how a state changes over time. The state collects the variables needed to describe where the system is right now, and the dynamics tell us how that point moves next.

For a pendulum, a useful state is the pair [position, velocity], or more specifically [theta, d theta / dt]. If we know those values, then a gravitational rule specifies how the pendulum should evolve at the next moment.

State

The state is the point in phase space. For this pendulum, that point is its position and velocity at a given instant.

Trajectory

A trajectory is the path traced out through state space as the system evolves from one initial condition.

Flow field

A flow field summarizes the local update rule by showing which direction the state tends to move from nearby points.

Ring attractor intuition

When energy is approximately conserved, trajectories keep circling around a closed orbit instead of collapsing to a point.

Spiral attractor intuition

With damping, nearby states lose energy and spiral into a stable fixed point rather than remaining on a closed loop.

0.1 What is the state?

In a dynamical system, the state should be compact but sufficient. For a pendulum, position alone is not enough, because the future depends on whether the pendulum is currently moving upward or downward. Adding velocity resolves that ambiguity.

0.2 Why use a phase portrait?

Plotting trajectories in state space makes the rule easier to see. Different initial conditions generate different paths, but they all follow the same underlying geometry. That geometry is often more informative than looking at position alone as a function of time.

0.3 Why estimate the flow field from observations?

In neural recordings we usually do not know the true governing equations directly. Instead, we observe state trajectories and try to infer the local rule that would produce them. The adaptive arrow field in the demo is a simple version of that idea: each new trajectory contributes evidence about how nearby states evolve.

A flow field is the underlying representation that tells each state which direction it should move next. If a system is unknown, we can still recover that structure by collecting trajectories in latent space, estimating the local vectors, and then asking what governing equations would be consistent with the recovered field.

In practice, that is the bridge from observation to mechanism: first measure trajectories, then infer the flow field, and finally propose or fit equations whose vector field reproduces the observed geometry.

Module 1 - Why dynamics matter for neural data

A central lesson from Mante et al. is that flexible behavior can be easier to understand at the population level than at the level of a single neuron. In their study, many prefrontal neurons showed mixed selectivity: the same cell could carry traces of sensory input, context, and impending choice. That makes raw single-neuron tuning hard to interpret in isolation, even when the population as a whole performs a clean computation.

Their state-space analysis used linear regression to define axes tied to major task variables, including choice. Once activity is projected onto those axes, the population trajectory becomes much clearer: irrelevant inputs can stay compressed, relevant inputs can separate, and the network can move toward a low-dimensional choice manifold over time.

A useful way to think about this is:

  1. Each moment of population firing defines one point in a high-dimensional neural state space.
  2. Across time, that point traces a trajectory.
  3. Regression or dimensionality-reduction methods can recover axes that align with task variables such as context, stimulus, and choice.
  4. The geometry of those trajectories often reveals the computation more directly than the tuning of any one neuron.

Mixed selectivity

A single neuron can reflect several variables at once, so its response may not map cleanly onto one task label.

Regression axes

Regressing population activity onto task variables can recover interpretable axes such as relevant input, context, or choice.

Population trajectory

Once the state is projected onto those axes, the evolving trajectory can reveal when evidence is selected, transformed, and committed to a choice.

1.1 A simple neuron-level example

Imagine a context-dependent task with two sensory streams, motion and color. On each trial the network is told which stream matters. In a small recurrent population, one neuron might respond strongly to rightward motion, another might respond to color but only when the color context is active, and a third might ramp with the eventual choice regardless of whether that choice was driven by motion or color. None of those neurons alone fully tells you the computation. The population trajectory does.

The toy network below is a simplified version of that logic. It is much smaller than the model in Mante et al., but it captures the same qualitative point: single units are mixed, recurrent interactions shape the state over time, and a regression-defined choice axis makes the emerging decision easier to see.

Coding Project 2.1

Integrate a pendulum from its state equation

Write the right-hand side of the pendulum system, then simulate trajectories from several initial angles. Start with zero initial velocity to match the demo. 

import numpy as np

g = 9.81
L = 1.0

def pendulum_rhs(theta, omega):
    dtheta = omega
    domega = -(g / L) * np.sin(theta)
    return dtheta, domega

def euler_step(theta, omega, dt):
    dtheta, domega = pendulum_rhs(theta, omega)
    return theta + dt * dtheta, omega + dt * domega

Coding Project 2.2

Estimate a flow field from sampled trajectories

Pretend the equations are unknown. Use the observed points and local finite differences to estimate the vector field over a grid in phase space. 

import numpy as np

def estimate_local_flow(points, derivatives, query, sigma=(0.7, 0.5)):
    dx = (points[:, 0] - query[0]) / sigma[0]
    dy = (points[:, 1] - query[1]) / sigma[1]
    weights = np.exp(-0.5 * (dx**2 + dy**2))

    if weights.sum() < 1e-6:
        return np.array([np.nan, np.nan])

    weighted = derivatives * weights[:, None]
    return weighted.sum(axis=0) / weights.sum()

Coding Project 2.3

Replace the pendulum with a simple neural latent system

Swap in a two-dimensional linear recurrent model and inspect how its trajectories differ from the pendulum. Then ask how a latent neural state might be interpreted in the same framework. 

import numpy as np

A = np.array([
    [0.0,  1.0],
    [-0.35, -0.18],
])

def latent_rhs(x):
    return A @ x

def rk4_step(x, dt):
    k1 = latent_rhs(x)
    k2 = latent_rhs(x + 0.5 * dt * k1)
    k3 = latent_rhs(x + 0.5 * dt * k2)
    k4 = latent_rhs(x + dt * k3)
    return x + (dt / 6.0) * (k1 + 2*k2 + 2*k3 + k4)
